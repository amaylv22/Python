Where do the Python unit tests go?

By convention all the tests for a module modulename.py should go in a file test_modulename.py, this file tests all of the functionality in the module.

All the test_*.py files in a directory name called tests/ at the root level of your package as a general rule.

\packagename
	__init__.py
	\tests
		test_modulename.py

Running tests

Since the test modules are in a separate directory from your code, you may need to add your moduleâ€™s parent directory to your PYTHONPATH in order to run them. In the ~/.bashrc and ~/.profile file add:

export PYTHONPATH=$PYTHONPATH:/path/to/your/dir

Open a new terminal and confirm the addition with:

$PYTHONPATH

Unit testing framework

There is one popular unit testing framework for Python, nose. nose helps simplify and extend the builtin unittest framework (it can, for example, automagically find your test code and setup your PYTHONPATH for you).
